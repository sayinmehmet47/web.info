How does the Internet works?

It works by using a packet routing network that follows Internet Protocol (IP) and Transport Control Protocol (TCP) . TCP and IP work together to ensure that data transmission across the internet is consistent and reliable, no matter which device you’re using or where you’re using it.

Internet Protocols

Common Internet protocols include TCP/IP (Transmission Control Protocol/Internet Protocol), UDP/IP (User Datagram Protocol/Internet Protocol), HTTP (HyperText Transfer Protocol) and FTP (File Transfer Protocol). TCP/IP is a stream protocol.

IP Addresses

In its simplest form, it is the identification number of our devices. An IP address consists of four numbers separated by dots; for example, your IP address is 00.00.000.000 ”. IP Address allows a computer on the Internet to be easily found by other devices. The websites we browse on the Internet are connected to a server. The IP addresses of the computers we call servers direct us to the site. So if you know the IP address of a site, you can connect to the website by typing this address in your browser. However, because this sequence of numbers is difficult to remember, there is a domain name for every IP address.

DNS

DNS is a system used to translate host names to IP addresses.Host names are called 'fully qualified name', which is a fully defined name. The host name shows the name of the computer and the internet domain (domain name) on your computer. In other words, DNS is a system consisting of resolvers and name servers.

Packets (TCP/UDP)

First, let's answer the question of what is a TCP port. TCP is short for Transmission Control Protocol. It is a network protocol that enables your data to be transmitted from your device to a server on the web. The answer to the question of what is a UDP port is similar: UDP stands for User Datagram Protocol and does exactly the same thing. In other words, both of these are the names of the network protocols. You use one of these protocols while surfing the internet, watching movies online, checking your emails. It transmits both your TCP and UDP data in small pieces known as "data packets". These packets contain sender and receiver IP addresses, different settings, the data you send and another data called "trailer" that indicates that the end of the packet has been reached. The only difference between the two protocols is in the way data packets are transported.

HTTP

In fact,in its simplest form, it is the protocol that allows web pages to be viewed. HTTP determines the rules of data exchange between the user's computer and the server. The browser is used to use this protocol. Google Chrome, Mozilla Firefox, Internet Explorer are some of these web browsers. With the help of these browsers, when you type the address of the site in the address bar to enter any website, a request is sent to the server via HTTP and when the server responds to this request, the data of the website comes to you. So you enter the website.

HTML

The basic concept that almost everyone who researches on web design comes across is HTML. HTML stands for "Hyper Text Markup Language", which is a standard text sign language used in the creation of web pages. Contrary to the common misconception, HTML is not a programming language. To put it more clearly, HTML is the language that browsers such as Chrome, Firefox, Yandex read and interpret.

Routers

Router, also known as Router, is the network hardware that provides the connection between multiple networks. That is, it allows the computers connecting to the network to be directed. Examines the incoming network packet and transfers it to another network.
